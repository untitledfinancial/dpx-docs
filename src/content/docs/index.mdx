## The moat

Every major stablecoin rail — Bridge, BVNK, Tazapay, Zero Hash — moves money. They are fast, cheap, and well-capitalised. DPX does something none of them do: it reasons about whether the conditions are right before every settlement, documents the ESG impact of every transaction on-chain, and is the only rail structured for AI agents to interact with natively.

That distinction compounds. Every settlement DPX processes is scored against its own predictions. Accuracy data accumulates. Signal weights recalibrate weekly. The oracle that runs today is more precise than the one that ran last month — and no competitor can replicate that history without running the same system for the same amount of time.

**Three things make this a durable moat:**

**1. Intelligence that changes outcomes, not just informs them.**
The 9-layer Stability Oracle produces STABLE / CAUTION / UNSTABLE signals with 30–90 day early warning. When the oracle flags CAUTION — yield curve inverting, shipping disruption, FX stress converging — treasury teams and agents hold large settlements. That is a risk management function with a measurable dollar value, not a dashboard feature.

**2. ESG compliance built into every transaction.**
CSRD in Europe now requires large companies to report sustainability impacts across their operations — including treasury and payments. SFDR requires financial institutions to disclose ESG exposure. DPX makes every settlement automatically ESG-documented with on-chain proof, verified by six institutional sources, and immutably recorded on-chain. The question to ask any European treasury team: *how are you currently documenting the ESG impact of your cross-border settlements for CSRD reporting?* DPX answers that question automatically.

**3. The only rail AI agents can reason about.**
Bridge and BVNK are built for humans using dashboards. DPX is built for agents making decisions. Every API response includes structured reasoning, a confidence score, and a forward-looking outlook — not just a price. Agents discover the protocol at `/manifest`, request a binding fee quote, verify terms on-chain, and execute settlement end-to-end without human approval. The MCP integration, GPT Actions schema, and LangChain tool wrappers are native — not bolted on.

We created a system that teaches itself. Every oracle run is scored against its own predictions. Every outcome is logged, measured, and fed back into the model. Signal weights adjust on a weekly cycle. Confidence calibrates over time. The longer DPX runs, the more precisely it understands which conditions matter most — and the more accurately it prices them. There is no fixed version of DPX. There is only the version running now, which is always more informed than the one before it.

```
GET /openapi.json                   OpenAPI 3.0 schema for all endpoints
GET /.well-known/ai-plugin.json     Agent framework plugin manifest
GET /manifest                       Live DPX protocol manifest with fee config
```
