No signup. No API key. Add `?sandbox=true` to any settlement call and the full stack runs — compliance preflight, fee math, oracle gating — without touching the chain.

---

## Run your first settlement

```bash
curl -X POST "https://integration.untitledfinancial.com/payments/initiate?sandbox=true" \
  -H "Content-Type: application/json" \
  -d '{
    "amount": "500000",
    "currency": "USD",
    "creditor": {
      "name": "Acme Logistics GmbH",
      "lei": "529900T8BM49AURSDO55",
      "walletAddress": "0xe333551E18ef0471A71d7e8e761212766aa5AD4f"
    },
    "debtor": {
      "name": "Acme Corp",
      "lei": "5493001KJTIIGC8Y1R12"
    },
    "purpose": "Supplier payment Q2 2026"
  }'
```

That's it. No token, no account. The response includes a settlement reference, full compliance result, fee breakdown, and ESG redistribution plan.

## What runs in sandbox mode

Everything except the on-chain transaction.

**Compliance preflight** — Verification of Payee confirms the creditor entity name. AML behavioural screening runs three layers: sanctions check, z-score anomaly detection, graph-based counterparty risk. Same logic as production, same latency.

**Fee calculation** — Live oracle math. Core settlement fee, FX overlay if cross-currency, ESG fee based on counterparty score. The number you see is the number production would charge.

**Oracle gating** — The Stability Oracle checks current conditions before every settlement. If the oracle is `UNSTABLE`, the sandbox call holds exactly as production would.

**ESG scoring** — The counterparty business is scored against 6 institutional sources. Score feeds into the fee. 100% of the ESG fee component is redistributed on-chain in production.

## The full sequence

If you want to see each layer individually before running settlement:

```bash
# 1. Protocol discovery — capabilities, contracts, corridors
curl https://stability.untitledfinancial.com/manifest

# 2. Binding fee quote — real oracle math, valid 300s
curl "https://stability.untitledfinancial.com/quote?amountUsd=500000&hasFx=true&esgScore=75"

# 3. Oracle stability check
curl https://stability.untitledfinancial.com/reliability

# 4. Sandbox settlement — full stack, no chain
curl -X POST "https://integration.untitledfinancial.com/payments/initiate?sandbox=true" \
  -H "Content-Type: application/json" \
  -d '{ "amount": "500000", "currency": "USD", "creditor": { "name": "Acme Logistics GmbH", "walletAddress": "0xe333551E18ef0471A71d7e8e761212766aa5AD4f" }, "debtor": { "name": "Acme Corp" } }'
```

Steps 1–3 have always been open. Step 4 is new — no auth required in sandbox mode.

## Sandbox vs production

| | Sandbox | Production |
|---|---|---|
| Auth required | No | Yes — institution key |
| Fee math | Live oracle | Live oracle |
| Compliance preflight | Full | Full |
| Oracle gating | Full | Full |
| On-chain transaction | No | Base mainnet |
| Settlement reference | `sandbox_` prefixed UUID | On-chain tx hash |
| ESG redistribution | Simulated | On-chain to impact pools |
