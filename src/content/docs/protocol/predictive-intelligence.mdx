## Regime Transition Probabilities

**Endpoint:** `GET /v1/intelligence/transition-risk`

Synthesises seven lead signals — climate, tectonic, water-risk, gender-risk, earth-systems, resonance, mycelium — into a current composite stress index and forward probability distributions over five regimes at 30, 60, and 90-day horizons.

**Five regimes:**

| Regime | Composite score band |
|---|---|
| `STABLE` | 0–20 |
| `ELEVATED` | 20–40 |
| `STRESSED` | 40–60 |
| `CRITICAL` | 60–80 |
| `CRISIS` | 80–100 |

**How horizon probabilities are calculated:**

Each horizon uses Gaussian kernel density estimation over the composite stress index, centred on the current value but with a widening standard deviation that reflects uncertainty at longer look-ahead distances. An amplification factor (derived from the breadth of divergence across signals) shifts the distribution toward higher-stress regimes when signals disagree. A transition signal (`DETERIORATING` / `STABLE` / `STABILIZING`) derived from the short-term vs long-term stress trajectory applies a directional bias.

Lead signal weights are horizon-adjusted using a decay function: signals with longer natural lead times (climate, tectonic, water-risk) are weighted more heavily at the 90-day horizon; signals with shorter lead times (resonance, mycelium) are weighted more heavily at 30 days. This means the 90-day forecast is climate-dominated; the 30-day forecast is financial-system-dominated.

## Prediction Ledger

**Endpoint:** `GET /v1/intelligence/ledger`

Every fresh (non-cache-hit) transition-risk call stores three predictions in the D1 prediction ledger — one for each horizon (30, 60, 90 days). Each row captures:

- The predicted regime and its probability at call time
- Full probability distribution over all five regimes
- Current composite score and regime at time of prediction
- Per-signal scores

At 02:00 UTC daily, a cron job resolves predictions whose horizons have arrived. It reads the current actual state and scores each resolved prediction using a proper probabilistic scoring rule. Scores accumulate in a calibration table per horizon (30d / 60d / 90d) and feed a weekly recalibration pass that corrects systematic over- or under-confidence in the raw probability outputs.

## Scenario Memory

Scenario memory runs inside every fresh transition-risk call. It is not a separate endpoint — it surfaces as the `scenarioMemory` field in the transition-risk response.

**How it works:**

Each transition-risk call builds a 32-dimensional vector encoding the full signal configuration, horizon probabilities, amplification, and regime distribution. This vector is stored in Cloudflare Vectorize under a timestamped ID with metadata (date, regime, score, per-signal scores, prediction IDs).

On the next call, the current signal vector is queried against all stored vectors using cosine similarity. The five closest historical configurations are returned with their similarity scores, stored regimes, and (once resolved) what regime they actually transitioned to.

Each vector encodes the full signal configuration: the seven lead signal scores, current regime state, horizon probability distributions at 30/60/90 days, amplification and transition signal, tail risk metrics, and derived signal statistics. The specific encoding is proprietary.

Similarity above 0.95 indicates the current configuration has occurred before at near-identical levels. The `patternNote` field translates the match into plain language: most common precedent regime, average similarity, and count.

As the index grows, this gives the oracle a genuine pattern library — the ability to say "this configuration appeared 3 times before; in 2 of 3 cases conditions deteriorated to CRITICAL within 60 days."

## Causal Graph

**Endpoint:** `GET /v1/intelligence/causal-graph`

A directed weighted graph of signal influence. 17 nodes across four tiers. Edge weights start from theoretical economic priors, then calibrate toward empirically observed signal relationships as daily observations accumulate.

**17 nodes:**

| Tier | Nodes |
|---|---|
| Lead (long lag) | `climate`, `water-risk`, `tectonic-structural`, `gender-risk` |
| Amplifier | `earth-systems`, `resonance`, `biodiversity`, `sovereign-debt` |
| Coincident | `mycelium`, `macro-stress`, `currency-stress`, `supply-chain`, `commodity` |
| Lagging | `energy-transition`, `instability`, `cascade`, `shipping-stress` |

**How edges calibrate:**

At 02:00 UTC daily, the cron reads the last 90 days of signal observations and computes empirical signal correlations at the natural lag for each edge pair. These are blended with the theoretical prior weights using a calibration factor that increases as more observations accumulate — starting from pure theory, transitioning toward empirical data as observations grow.

With fewer than 10 observations, the graph is in `PRIOR` phase — edges reflect economic theory only. At 10–50 observations, it is in `BLENDING` phase — empirical data begins to take over. Above 50 observations, it reaches `EMPIRICAL` phase — the graph is predominantly data-driven.

**Prior edges seeded from economic literature:**

Directional relationships between nodes are initialised from established macroeconomic and financial theory — for example, climate stress leading water risk and supply chain pressure at longer lags, macro stress leading currency stress at shorter lags. The full edge set and weights are proprietary.

## How the four systems work together

```
Every fresh call to /transition-risk:
  1. Fetches 7 signal scores in parallel (20s timeout each)
  2. Computes composite index + horizon probability distributions
  3. Stores 3 predictions in D1 prediction ledger (h30 / h60 / h90)
  4. Builds 32-dim vector → upserts to Vectorize
  5. Queries Vectorize for 5 most similar historical configurations
  6. Returns transition-risk result with scenarioMemory attached

Every day at 02:00 UTC:
  1. Reads current actual state from KV cache (transition-risk)
  2. Resolves any predictions whose horizons have arrived
  3. Scores resolved predictions, updates calibration table
  4. Reads 90d signal observation history from D1
  5. Recomputes empirical signal correlations per edge
  6. Updates causal edge weights (blend theory + empirical)
```

The result is a system that improves with time in three distinct ways: probabilistic calibration improves as forecast accuracy accumulates and probability outputs are recalibrated; pattern matching improves as the scenario memory index grows; and causal understanding improves as empirically observed signal relationships gradually displace theoretical prior weights.

## Calibration phases

| Phase | Observations | Behaviour |
|---|---|---|
| `PRIOR` | < 10 | All edge weights from economic literature. Predictions uncalibrated. |
| `BLENDING` | 10–50 | Empirical data takes increasing share of edge weights. Probability recalibration begins with limited sample. |
| `EMPIRICAL` | 50+ | Edge weights predominantly empirical. Probability recalibration operating on meaningful sample size. |

The system enters BLENDING phase after approximately 10 daily cron cycles (~10 days of live operation). It reaches full EMPIRICAL calibration after approximately 50 daily cycles.
